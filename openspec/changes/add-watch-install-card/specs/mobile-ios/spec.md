## ADDED Requirements

### Requirement: 설정 화면 최상단 워치 앱 상태 카드

iOS 폰 앱의 설정 화면은 화면 타이틀("설정") 바로 다음, 다른 어떤 설정 항목보다 위에 워치 동반 앱 상태 카드를 노출해야 한다(SHALL). 카드는 워치 동반 앱 상태에 따라 다음과 같이 표현되어야 한다:

- **PAIRED_NONE**: 회색 톤 카드 + "Apple Watch를 페어링해 주세요" 안내. 설치 버튼은 비활성 또는 숨김.
- **PAIRED_NO_APP**: 워치 아이콘 위 빨간 경고 배지 + "워치 앱 설치 필요" 제목 + 자동 설치 옵션 안내 + [워치 앱 설치하기] 기본 버튼 + [연결 확인] 보조 버튼.
- **INSTALLED**: 초록 체크 + "워치 앱 설치됨" 제목 + 컴팩트한 [다시 확인] 보조 버튼. 카드 높이는 다른 두 상태보다 작아야 한다.

카드의 기본 버튼 색상은 사용자가 선택한 응원 팀 컬러를 따라야 한다(SHALL). 디자인 토큰과 카드 구조는 Android 와 시각적으로 일관되어야 한다.

#### Scenario: Apple Watch를 페어링하지 않은 사용자가 설정 탭을 열었을 때

- **WHEN** Apple Watch가 폰에 페어링되어 있지 않은 사용자가 설정 탭에 진입
- **THEN** "설정" 타이틀 바로 아래에 PAIRED_NONE 상태 카드가 가장 먼저 표시되고, Apple Watch 페어링 안내 문구가 노출된다

#### Scenario: Apple Watch는 있지만 야구봄 워치 앱이 설치되지 않은 사용자가 설정 탭을 열었을 때

- **WHEN** Apple Watch는 페어링되었으나 야구봄 워치 앱이 설치되어 있지 않은 사용자가 설정 탭에 진입
- **THEN** "설정" 타이틀 바로 아래에 PAIRED_NO_APP 상태 카드가 가장 먼저 표시되고, [워치 앱 설치하기] 버튼이 응원 팀 컬러로 노출된다

#### Scenario: 워치 앱이 정상 설치된 사용자가 설정 탭을 열었을 때

- **WHEN** 야구봄 워치 앱이 설치된 사용자가 설정 탭에 진입
- **THEN** "설정" 타이틀 바로 아래에 INSTALLED 상태의 컴팩트 카드가 가장 먼저 표시되고, 초록 체크와 함께 "워치 앱 설치됨" 문구가 노출된다

### Requirement: Apple Watch 앱 설치 유도 흐름

PAIRED_NO_APP 상태에서 [워치 앱 설치하기] 버튼은 시스템의 Apple Watch 앱을 호출하여 사용자가 사용 가능한 앱 목록에서 야구봄 워치 앱을 설치할 수 있도록 안내해야 한다(SHALL).

iOS 의 동반 앱 자동 설치 옵션이 켜져 있는 경우 폰 앱 설치 시 자동으로 설치되므로, 카드에는 자동 설치가 일반적이며 본 버튼은 fallback 임을 안내하는 문구가 함께 노출되어야 한다.

#### Scenario: 사용자가 [워치 앱 설치하기] 버튼을 누름

- **WHEN** PAIRED_NO_APP 상태에서 사용자가 [워치 앱 설치하기] 버튼을 누름
- **THEN** 시스템 Apple Watch 앱이 열리고, 사용자가 사용 가능한 앱 목록에서 야구봄을 설치할 수 있다

#### Scenario: 시스템 Apple Watch 앱 호출이 동작하지 않는 환경

- **WHEN** 시스템 Apple Watch 앱 호출 URL 이 동작하지 않는 iOS 버전에서 사용자가 버튼을 누름
- **THEN** 시스템은 사용자에게 직접 Apple Watch 앱에서 설치하도록 안내하는 fallback 문구를 표시한다
