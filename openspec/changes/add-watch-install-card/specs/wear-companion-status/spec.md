## ADDED Requirements

### Requirement: 워치 동반 앱 상태 정의

시스템은 폰 앱 관점에서 워치 동반 앱의 상태를 다음 세 가지 중 정확히 하나로 분류해야 한다(SHALL):
- **페어링 안 됨(PAIRED_NONE)**: 사용자의 폰에 어떤 워치도 페어링되어 있지 않은 상태
- **페어링됨, 앱 미설치(PAIRED_NO_APP)**: 워치는 페어링되어 있으나 워치에 야구봄 앱이 설치되어 있지 않은 상태
- **설치됨(INSTALLED)**: 워치가 페어링되어 있고 워치에 야구봄 앱이 설치되어 있는 상태

상태 결정은 워치 앱의 실행(foreground) 여부와 무관해야 한다(MUST). 사용자가 워치 앱을 한 번도 켜지 않았어도 설치되어 있으면 INSTALLED 로 분류되어야 한다.

#### Scenario: 페어링된 워치가 없을 때

- **WHEN** 사용자가 어떤 워치도 폰과 페어링하지 않은 채 설정 화면에 진입
- **THEN** 시스템은 상태를 PAIRED_NONE 으로 보고한다

#### Scenario: 워치는 있지만 앱이 설치되지 않았을 때

- **WHEN** 사용자가 워치를 페어링한 상태에서, 그 워치에 야구봄 앱이 설치되어 있지 않을 때
- **THEN** 시스템은 상태를 PAIRED_NO_APP 으로 보고한다

#### Scenario: 워치 앱이 설치되어 있고 한 번도 실행되지 않았을 때

- **WHEN** 사용자가 워치에 야구봄 앱을 설치만 하고 단 한 번도 실행하지 않은 상태
- **THEN** 시스템은 상태를 INSTALLED 로 보고한다

### Requirement: 워치 앱 설치 유도 진입점

시스템은 PAIRED_NO_APP 상태에서 사용자가 워치 앱을 설치할 수 있는 진입점을 제공해야 한다(SHALL).

Android 플랫폼에서는 폰의 앱 마켓 상세 페이지로 이동시켜 사용자가 설치 대상 기기를 워치로 선택하도록 안내해야 한다. iOS 플랫폼에서는 시스템 워치 동반 앱(Apple Watch 앱)을 호출해 사용 가능한 앱 목록으로 사용자를 보내야 한다.

#### Scenario: Android 사용자가 설치 버튼을 누름

- **WHEN** Android 사용자가 PAIRED_NO_APP 상태에서 [워치 앱 설치하기] 버튼을 누름
- **THEN** 폰의 앱 마켓 상세 페이지가 열리고, 사용자가 설치 대상 기기 선택 드롭다운에서 워치를 선택할 수 있는 안내 문구가 카드에 함께 표시되어 있다

#### Scenario: iOS 사용자가 설치 버튼을 누름

- **WHEN** iOS 사용자가 PAIRED_NO_APP 상태에서 [워치 앱 설치하기] 버튼을 누름
- **THEN** 시스템 Apple Watch 앱이 열리고, 사용자가 사용 가능한 앱 목록에서 야구봄 앱을 설치할 수 있다

### Requirement: 상태 갱신 트리거

시스템은 다음 상황에서 워치 동반 앱 상태를 재조회해야 한다(SHALL):
- 사용자가 카드가 포함된 화면에 진입할 때 1회
- 카드가 포함된 화면이 백그라운드에서 포그라운드로 복귀할 때
- 사용자가 [연결 확인] 또는 [다시 확인] 버튼을 누를 때

배터리 보호를 위해 자동 폴링을 수행해서는 안 된다(MUST NOT).

#### Scenario: 사용자가 설정 화면을 처음 열 때

- **WHEN** 사용자가 설정 탭에 진입
- **THEN** 시스템은 워치 동반 앱 상태를 1회 조회하고 카드에 결과를 반영한다

#### Scenario: 사용자가 워치 앱 설치를 완료하고 폰 앱으로 돌아왔을 때

- **WHEN** 사용자가 [워치 앱 설치하기] 버튼으로 외부 앱에 다녀온 뒤 폰 앱이 포그라운드로 복귀
- **THEN** 시스템은 자동으로 상태를 재조회하여 INSTALLED 로 갱신한다

#### Scenario: 사용자가 [연결 확인] 버튼을 누름

- **WHEN** 사용자가 카드의 [연결 확인] 또는 [다시 확인] 버튼을 누름
- **THEN** 시스템은 즉시 상태를 재조회하고 결과를 카드에 반영한다

### Requirement: 워치 앱 실행 강제 금지

상태 감지 로직은 워치 앱이 사용자의 손목에서 실행되도록 강제하거나 그것을 전제로 해서는 안 된다(MUST NOT). 푸시 알림 기반 동작을 사용하는 사용자가 워치 앱을 평소에 켜놓지 않더라도 INSTALLED 상태로 정확히 표시되어야 한다.

#### Scenario: 워치 앱이 백그라운드/대기 상태일 때

- **WHEN** 워치 앱이 설치되어 있으나 현재 워치 화면에 표시되어 있지 않은 상태
- **THEN** 시스템은 상태를 INSTALLED 로 보고하며, "연결 안 됨" 으로 잘못 표시하지 않는다
