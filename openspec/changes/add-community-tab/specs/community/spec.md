## ADDED Requirements

### Requirement: 팀별 오픈 채팅방
시스템은 KBO 구단별로 오픈 채팅방을 1개씩 제공해야 한다(SHALL). 파일럿 단계에서는 LG 트윈스·두산 베어스 2팀만 방을 활성화하고, 나머지 팀은 "곧 오픈" 상태로만 노출한다.

#### Scenario: 로그인 사용자가 내 응원팀 방에 입장
- **WHEN** 사용자가 커뮤니티 홈에서 자신의 응원팀 카드를 누른다
- **THEN** 해당 팀의 오픈 채팅방에 입장해 메시지를 읽고 작성할 수 있다

#### Scenario: 로그인 사용자가 타팀 방을 방문
- **WHEN** 사용자가 커뮤니티 홈에서 응원팀이 아닌 팀의 방을 누른다
- **THEN** 해당 방의 메시지를 읽을 수 있으나, 메시지 작성은 차단된다(파일럿 옵션 A)

#### Scenario: 비활성 팀 방 노출
- **WHEN** 파일럿 대상이 아닌 팀 카드가 화면에 표시된다
- **THEN** "곧 오픈" 라벨과 함께 회색 비활성 상태로 표시되고 탭할 수 없다

### Requirement: 텔레그램식 답장 스레드
시스템은 기사에 직접 댓글을 달 수 없고(MUST NOT), 대신 팀 채팅방의 메시지에 답장하는 방식으로만 대화를 구성해야 한다(SHALL). 봇이 새 기사를 감지하면 해당 팀방에 핀 메시지로 포스팅한다.

#### Scenario: 기사가 방에 자동 포스팅된다
- **WHEN** 크롤러가 LG 관련 신규 기사를 수집한다
- **THEN** LG 팀방에 해당 기사가 핀 메시지 카드로 자동 포스팅된다

#### Scenario: 사용자가 기사에 답장한다
- **WHEN** 사용자가 핀 기사 메시지를 길게 눌러 "답장"을 선택하고 내용을 입력한다
- **THEN** 새 메시지가 해당 기사에 대한 답장으로 스레드에 연결된다

#### Scenario: 메시지에 대한 답장
- **WHEN** 사용자가 다른 사용자의 메시지에 답장한다
- **THEN** 새 메시지는 원본 메시지를 인용하는 형태로 표시된다

### Requirement: 메시지 단위 익명 토글
시스템은 각 메시지 작성 시 "익명" 토글을 제공해야 한다(SHALL). 한 사용자가 한 방 안에서는 익명 상태를 on/off 혼용할 수 없고, 방 단위로 고정된다.

#### Scenario: 익명으로 메시지 작성
- **WHEN** 사용자가 메시지 입력창에서 익명 토글을 켜고 메시지를 보낸다
- **THEN** 다른 사용자에게는 작성자가 "익명" 아바타로 표시된다

#### Scenario: 같은 방 안 익명 고정
- **WHEN** 사용자가 한 방에서 이미 실명으로 메시지를 보낸 뒤 같은 방에서 익명으로 전환하려 한다
- **THEN** 시스템은 해당 방에서 기존 표시 방식을 유지하며 전환을 차단한다

#### Scenario: 내부 user_id 추적 유지
- **WHEN** 메시지가 익명으로 작성된다
- **THEN** 내부 데이터베이스에는 실제 user_id가 저장되어 법적 요청 시 추적 가능하다

### Requirement: 🔥 핫 기사 피드
시스템은 커뮤니티 홈 상단에 전 구단 통합 핫 기사 피드를 가로 스크롤 카드로 노출해야 한다(SHALL). 사용자는 타팀 대화도 가시적으로 확인할 수 있다.

#### Scenario: 핫 기사 랭킹 산출
- **WHEN** 시스템이 핫 기사 점수를 계산한다
- **THEN** 점수는 고유 참여자 수·답장 수·좋아요·신고 수를 가중합한 뒤 시간감쇠를 적용한 값을 사용한다

#### Scenario: 핫 카드에서 기사 스레드 진입
- **WHEN** 사용자가 핫 기사 카드를 탭한다
- **THEN** 해당 기사의 스레드 화면이 열려 "이 기사 답장" 탭이 기본 선택된 상태로 표시된다

#### Scenario: 신고 누적 기사 자동 숨김
- **WHEN** 핫 기사가 누적 신고로 모더레이션 임계치를 넘는다
- **THEN** 해당 기사는 핫 기사 피드에서 자동으로 제외된다

### Requirement: 기사 스레드 화면
시스템은 기사 상세 진입 시 "이 기사 답장" 탭과 "전체 방 대화" 탭을 제공해야 한다(SHALL).

#### Scenario: 이 기사 답장 탭
- **WHEN** 사용자가 "이 기사 답장" 탭을 선택한다
- **THEN** 해당 기사에 직접 연결된 답장 메시지만 시간순으로 표시된다

#### Scenario: 전체 방 대화 탭
- **WHEN** 사용자가 "전체 방 대화" 탭을 선택한다
- **THEN** 해당 팀의 오픈 채팅방 전체 메시지 스트림이 표시되며, 선택한 기사는 상단 핀으로 고정된다
