## ADDED Requirements

### Requirement: 기사 메타데이터 수집
시스템은 외부 뉴스 소스에서 기사 메타데이터(제목, 2~3줄 요약, 썸네일 URL, 원문 URL, 출처명, 발행 시각)만 수집해야 한다(SHALL). 기사 본문 전체는 저장하지 않는다(MUST NOT).

#### Scenario: 신규 기사 수집
- **WHEN** 크롤러가 외부 소스에서 새 기사를 발견한다
- **THEN** 시스템은 제목·요약·썸네일 URL·원문 URL·출처·발행 시각을 저장하고, 본문 전체는 저장하지 않는다

#### Scenario: 썸네일 재호스팅 금지
- **WHEN** 기사 썸네일이 수집된다
- **THEN** 시스템은 언론사가 제공하는 원본 이미지 URL만 참조하고, 이미지 파일을 자체 스토리지에 복제하지 않는다

#### Scenario: 원문 링크 필수 노출
- **WHEN** 사용자가 기사 상세를 본다
- **THEN** 시스템은 원문 보기 링크를 항상 노출해 외부 사이트로 이동할 수 있게 한다

### Requirement: 팀 태깅 및 중복 제거
시스템은 수집한 기사를 KBO 구단별로 태깅하고(SHALL), 동일 기사의 중복 수집을 방지해야 한다(SHALL).

#### Scenario: 팀 태깅
- **WHEN** 새 기사가 수집된다
- **THEN** 시스템은 제목·요약·원문 URL을 기반으로 관련 구단을 식별해 태그를 부여한다

#### Scenario: 중복 수집 방지
- **WHEN** 같은 원문 URL의 기사가 두 번 이상 수집 대상이 된다
- **THEN** 시스템은 첫 수집만 반영하고 이후 중복을 무시한다

### Requirement: 기사 자동 포스팅
시스템은 수집한 기사를 해당 팀의 오픈 채팅방에 핀 메시지로 자동 포스팅해야 한다(SHALL).

#### Scenario: LG 기사 자동 포스팅
- **WHEN** 크롤러가 LG 태그가 부여된 새 기사를 저장한다
- **THEN** 시스템은 LG 팀방에 해당 기사를 핀 메시지로 자동 포스팅한다

#### Scenario: 비활성 팀 기사 보류
- **WHEN** 파일럿 대상이 아닌 팀의 기사가 수집된다
- **THEN** 시스템은 기사를 저장하지만 방이 열릴 때까지 포스팅하지 않는다
