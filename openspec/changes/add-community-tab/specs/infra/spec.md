## ADDED Requirements

### Requirement: Supabase Auth 소셜 공급자
인프라는 Supabase Auth에 카카오와 Apple 소셜 공급자를 등록·운영해야 한다(SHALL).

#### Scenario: 카카오 공급자 연동
- **WHEN** 사용자가 앱에서 카카오 로그인을 시도한다
- **THEN** Supabase Auth가 카카오 OAuth 흐름을 처리해 세션을 발급한다

#### Scenario: Apple 공급자 연동
- **WHEN** 사용자가 앱에서 Apple 로그인을 시도한다
- **THEN** Supabase Auth가 Apple Sign In 흐름을 처리해 세션을 발급한다

### Requirement: 실시간 채팅 브로드캐스트
인프라는 팀 채팅방의 메시지 생성·수정·블라인드 이벤트를 실시간으로 클라이언트에 전달해야 한다(SHALL).

#### Scenario: 신규 메시지 실시간 수신
- **WHEN** 한 사용자가 팀방에 메시지를 전송한다
- **THEN** 같은 팀방에 접속 중인 다른 사용자에게 즉시 새 메시지가 표시된다

#### Scenario: 자동 블라인드 실시간 반영
- **WHEN** 모더레이션 규칙에 의해 메시지가 자동 블라인드된다
- **THEN** 해당 팀방의 모든 접속 사용자 화면에서 메시지가 즉시 블라인드 플레이스홀더로 바뀐다

### Requirement: 기사 크롤러 스케줄
인프라는 기사 크롤러를 주기적으로 실행하고 결과를 데이터베이스에 저장해야 한다(SHALL).

#### Scenario: 정기 크롤링
- **WHEN** 크롤러 스케줄 시점이 도래한다
- **THEN** 시스템은 대상 뉴스 소스를 호출해 신규 기사 메타데이터를 수집·저장한다

### Requirement: 핫 기사 랭킹 집계
인프라는 핫 기사 랭킹을 주기적으로 재계산해 커뮤니티 홈에 반영해야 한다(SHALL). 재계산 주기는 5분을 초과하지 않는다.

#### Scenario: 랭킹 재계산
- **WHEN** 랭킹 집계 주기가 경과한다
- **THEN** 시스템은 모든 활성 팀방의 기사에 대해 점수와 순위를 다시 계산한다

### Requirement: 법적 요청 대응 데이터 보관
인프라는 익명 표시 여부와 무관하게 모든 메시지에 대해 작성자 user_id를 최소 법정 기간 동안 보관해야 한다(MUST).

#### Scenario: 법적 요청 대응
- **WHEN** 수사기관 등 적법한 요청이 접수된다
- **THEN** 시스템은 해당 메시지의 작성자 식별자를 제공할 수 있어야 한다
