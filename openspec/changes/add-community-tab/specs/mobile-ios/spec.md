## ADDED Requirements

### Requirement: 커뮤니티 탭 하단 내비게이션 추가
iOS 앱은 하단 탭바에 커뮤니티 탭을 추가해야 한다(SHALL). 탭바 구성은 HOME / LIVE / COMMUNITY / ME 순서를 따른다.

#### Scenario: 커뮤니티 탭 표시
- **WHEN** 사용자가 앱을 실행해 하단 탭바를 본다
- **THEN** HOME, LIVE, COMMUNITY, ME 4개 탭이 순서대로 표시되고 COMMUNITY 탭이 활성 상태일 때 강조 스타일이 적용된다

### Requirement: 커뮤니티 홈 화면
iOS 앱은 커뮤니티 홈 화면에서 🔥 핫 기사 카드 가로 스크롤, 내 팀 라운지 큰 카드, 다른 팀 리스트를 위에서 아래로 표시해야 한다(SHALL).

#### Scenario: 커뮤니티 홈 진입
- **WHEN** 로그인한 사용자가 커뮤니티 탭을 선택한다
- **THEN** 상단에 핫 기사 가로 카드, 중앙에 내 응원팀 큰 카드, 아래에 다른 팀 리스트가 표시된다

### Requirement: 팀 채팅방 화면
iOS 앱은 팀 채팅방 화면에서 상단 팀 정보 헤더, 핀 기사 영역, 메시지 리스트, 하단 입력바(익명 토글 포함)를 제공해야 한다(SHALL).

#### Scenario: 채팅방 입장
- **WHEN** 사용자가 내 팀 라운지 카드를 누른다
- **THEN** 팀명과 접속자 수가 상단에, 최근 핀 기사가 그 아래에, 메시지 리스트와 입력바가 화면 하단에 배치된다

### Requirement: 기사 스레드 상세 화면
iOS 앱은 기사 상세 화면에서 기사 카드, 원문 보기 버튼, "이 기사 답장 / 전체 방 대화" 탭, 메시지 스레드를 제공해야 한다(SHALL).

#### Scenario: 핫 카드에서 스레드 진입
- **WHEN** 사용자가 커뮤니티 홈에서 핫 기사 카드를 누른다
- **THEN** 기사 스레드 화면이 "이 기사 답장" 탭이 기본 활성화된 상태로 표시된다

### Requirement: 로그인 화면
iOS 앱은 커뮤니티 탭 최초 진입 시 로그인 화면을 표시해야 하며(SHALL), 화면에는 카카오 버튼과 Apple 버튼만 제공한다.

#### Scenario: 미로그인 상태 커뮤니티 진입
- **WHEN** 로그인하지 않은 사용자가 커뮤니티 탭을 누른다
- **THEN** 로그인 화면이 표시되고 카카오·Apple 버튼 외에는 노출되지 않는다

### Requirement: 신고 모달
iOS 앱은 메시지 신고를 바텀시트 형태로 제공해야 한다(SHALL). 사유 5종 라디오, 상세 입력 텍스트필드, 제출 버튼, 24시간 검토 안내를 포함한다.

#### Scenario: 메시지 신고
- **WHEN** 사용자가 메시지를 길게 눌러 신고를 선택한다
- **THEN** 신고 사유 5종이 나열된 바텀시트가 나타나고 제출 후 "24시간 내 검토" 안내가 표시된다

### Requirement: 다크 모드 우선 구현
iOS 커뮤니티 화면 5종은 본 차수에서 다크 모드만 구현해야 한다(SHALL). 라이트 모드는 제외한다.

#### Scenario: 다크 모드 렌더링
- **WHEN** 사용자가 커뮤니티 화면에 진입한다
- **THEN** 시스템 설정과 무관하게 다크 팔레트가 적용된다
