## Why

야구봄은 현재 실시간 중계·햅틱 피드백에 집중돼 있어 팬들이 경기 전후·이적 루머·기사 해석 같은 "대화" 욕구를 풀 공간이 없다. 네이버·다음이 스포츠 기사 댓글을 폐지한 뒤 팬들은 흩어진 오픈채팅·디시인사이드·엑스(트위터)에서 대화를 이어가는데, 법적 리스크(초상권·명예훼손 방조 책임)와 모더레이션 부담을 감당할 수 있는 구조로 "야구봄 안의 커뮤니티"를 만들면 실시간 중계 기능과 자연스럽게 이어져 체류시간과 리텐션을 동시에 끌어올릴 수 있다.

## What Changes

- 하단 탭바에 **커뮤니티 탭** 추가 (HOME/LIVE/**COMMUNITY**/ME)
- 카카오·애플 **소셜 로그인** 도입 (기존 앱은 로그인 없이 사용 가능했음 — **BREAKING** 사용자 프로필/인증 상태 관리 신규 도입)
- **팀별 오픈 채팅방** 구조 (LG·두산 2팀 파일럿, 나머지 "곧 오픈")
- **텔레그램식 답장 스레드** — 기사에 직접 댓글 X, 대신 기사는 방에 핀 메시지로 포스팅되고 사용자끼리 답장으로 대화
- 메시지 단위 **익명 토글** (방마다 on/off 고정, 가입은 실명 기반)
- **🔥 핫 기사 피드** — 전 구단 통합 랭킹으로 타팀 대화도 가시화
- **기사 크롤러** — 네이버 스포츠 등 외부 소스에서 제목+요약+원문 링크만 수집 (본문 복제 X)
- **모더레이션 파이프라인** — 금칙어 필터, AI 모더레이션(OpenAI Moderation API or Perspective), 신고 3회 자동 블라인드, 24시간 내 처리 SLA
- 커뮤니티 가이드라인·개인정보처리방침 동의 플로우

## Capabilities

### New Capabilities
- `community`: 팀별 오픈 채팅방, 메시지/답장 스레드, 핀 기사, 익명 토글, 🔥 핫 기사 피드, 방 입장·읽기 권한 정책
- `auth`: 카카오/애플 OAuth 로그인, 세션 관리, 사용자 프로필(닉네임·응원팀·프로필 이미지), 법적 요청 대응을 위한 user_id 추적
- `news-ingest`: 네이버 스포츠 등 기사 크롤러, 팀 태깅, 중복 제거, 원문 링크 저장 (본문 전체 복제 금지)
- `moderation`: 신고 시스템, 금칙어 필터, AI 모더레이션, 자동 블라인드 규칙, 24시간 SLA, 이용약관/가이드라인 관리

### Modified Capabilities
- `mobile-ios`: 커뮤니티 탭 화면 5개 추가 (홈·팀방·기사 스레드·로그인·신고 모달), 하단 탭바에 항목 추가
- `mobile-android`: 동일한 화면 세트 추가 (iOS와 대칭)
- `infra`: Supabase Auth(카카오·애플), Realtime 채널, Railway 크론(크롤러), 핫 기사 랭킹 뷰

## Impact

- **코드**: iOS/Android 신규 화면 5종, 백엔드 API 신규(rooms/messages/articles/reports/auth), 크롤러 신규, DB 마이그레이션(신규 테이블 7~9개)
- **의존성**: Supabase Auth 공급자 설정(Kakao·Apple), OpenAI Moderation API 또는 Google Perspective API, 카카오 개발자 앱 등록, Apple Sign In 키
- **법무**: 이용약관·개인정보처리방침·커뮤니티 가이드라인 3종 초안 필요. 통신비밀보호법·정보통신망법 준수 체크.
- **디자인**: Pencil 와이어프레임 완료(`/pencil-new.pen`, 다크 모드 5화면). 라이트 모드는 본 차수 Non-Goal.
- **Non-Goals (이번 차수 제외)**:
  - 라이트 모드
  - 10개 구단 전체 확장 (LG/두산만)
  - 타팀 참여 정책 최종 결정(읽기만 vs 개방 vs 하이브리드) — 파일럿 데이터 보고 후속 차수에서 결정
  - 기사에 직접 댓글 기능
