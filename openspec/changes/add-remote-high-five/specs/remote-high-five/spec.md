## ADDED Requirements

### Requirement: 사용자 opt-in 토글
시스템은 "원격 하이파이브" 기능을 설정 화면 토글로 제공해야 하며(SHALL), 기본값은 꺼짐(off)이어야 한다. 토글이 켜질 때만 블루투스 관련 권한을 요청해야 하며(MUST), 토글이 꺼져 있는 동안에는 어떠한 BLE 광고나 스캔도 수행하지 않아야 한다(MUST NOT).

#### Scenario: 기본 상태는 꺼짐
- **WHEN** 사용자가 앱을 처음 설치하고 설정 화면에 진입한다
- **THEN** "원격 하이파이브" 토글이 꺼진 상태로 표시된다
- **AND** 시스템은 블루투스 권한을 요청하지 않는다

#### Scenario: 토글 on 시 권한 요청
- **WHEN** 사용자가 토글을 켠다
- **THEN** 시스템은 블루투스 권한 다이얼로그를 표시한다
- **AND** 권한 문구에는 "근처 같은 팀 팬을 감지하기 위해 사용하며, 위치 정보를 수집하지 않는다"는 의미가 포함된다

#### Scenario: 권한 거부 시 토글 복구
- **WHEN** 사용자가 권한을 거부한다
- **THEN** 토글은 자동으로 꺼진 상태로 되돌아간다
- **AND** 시스템은 BLE 광고·스캔을 시작하지 않는다

#### Scenario: 토글 off 즉시 중단
- **WHEN** 사용자가 토글을 끈다
- **THEN** 진행 중이던 모든 BLE 광고와 스캔이 즉시 중단된다

### Requirement: 이벤트 트리거 기반 근접 감지 윈도우
시스템은 평상시에 BLE 광고·스캔을 수행하지 않아야 하며(MUST NOT), 오직 사용자의 응원 팀이 득점 또는 홈런 이벤트를 수신한 순간에만 제한된 시간 동안 BLE 광고·스캔 윈도우를 열어야 한다(SHALL).

#### Scenario: 내 팀 득점 시 윈도우 열림
- **WHEN** 사용자의 응원 팀에 대한 득점 이벤트가 수신된다
- **THEN** 시스템은 BLE 광고와 스캔을 동시에 시작한다
- **AND** 윈도우는 20초 이내에 자동으로 닫힌다

#### Scenario: 내 팀 홈런 시 윈도우 열림
- **WHEN** 사용자의 응원 팀에 대한 홈런 이벤트가 수신된다
- **THEN** 시스템은 BLE 광고와 스캔을 시작한다

#### Scenario: 상대 팀 이벤트에는 윈도우 없음
- **WHEN** 상대 팀에 대한 득점/홈런 이벤트가 수신된다
- **THEN** 시스템은 BLE 광고·스캔을 시작하지 않는다

#### Scenario: 지연 수신 이벤트 무시
- **WHEN** 수신된 이벤트의 발생 시각이 현재 시각보다 60초 이상 과거이다
- **THEN** 시스템은 윈도우를 열지 않는다

#### Scenario: 평상시 광고·스캔 금지
- **WHEN** 이벤트 트리거 윈도우가 아닌 시간대이다
- **THEN** 시스템은 BLE 광고나 스캔을 수행하지 않는다

### Requirement: 익명 브로드캐스트 페이로드
시스템의 BLE 광고 페이로드는 익명이어야 하며(SHALL), 사용자나 기기를 재식별할 수 있는 식별자(계정 ID, 닉네임, 영구 UUID, 전화번호 등)를 포함해서는 안 된다(MUST NOT). 페이로드에는 응원 팀 식별자, 이벤트 종류, 1분 이내 주기로 회전하는 랜덤 토큰, 프로토콜 버전만 포함되어야 한다.

#### Scenario: 허용된 필드만 포함
- **WHEN** 시스템이 BLE 광고를 송신한다
- **THEN** 페이로드에는 팀 식별자, 이벤트 종류, 회전 토큰, 프로토콜 버전만 포함된다

#### Scenario: 개인 식별자 금지
- **WHEN** 시스템이 BLE 광고 페이로드를 구성한다
- **THEN** 계정 ID, 사용자 닉네임, 영구 UUID, 전화번호, 위치 좌표가 포함되지 않는다

#### Scenario: 토큰 회전
- **WHEN** 직전 회전 후 60초가 경과한다
- **THEN** 다음 광고 페이로드의 회전 토큰은 새 랜덤값으로 교체된다

### Requirement: 매칭 판정 규칙
시스템은 수신한 BLE 광고 중 같은 팀·같은 이벤트 종류이며 RSSI가 설정된 임계값 이상인 경우에만 "하이파이브 성립"으로 판정해야 한다(SHALL). 다른 팀 광고, 이벤트 종류가 다른 광고, 임계값 미만의 광고는 하이파이브로 처리해서는 안 된다(MUST NOT).

#### Scenario: 같은 팀·같은 이벤트·가까운 거리
- **WHEN** 수신 광고의 팀이 사용자 응원 팀과 같고 이벤트 종류가 현재 윈도우와 같으며 RSSI가 임계값 이상이다
- **THEN** 시스템은 해당 광고를 하이파이브 성립으로 판정한다

#### Scenario: 다른 팀 광고 무시
- **WHEN** 수신 광고의 팀 식별자가 사용자 응원 팀과 다르다
- **THEN** 시스템은 해당 광고를 무시한다

#### Scenario: 이벤트 종류 불일치 무시
- **WHEN** 수신 광고의 이벤트 종류가 현재 열린 윈도우의 이벤트 종류와 다르다
- **THEN** 시스템은 해당 광고를 무시한다

#### Scenario: 거리 임계값 미만 무시
- **WHEN** 수신 광고의 RSSI가 설정된 임계값 미만이다
- **THEN** 시스템은 해당 광고를 무시한다

#### Scenario: 동일 토큰 중복 집계 방지
- **WHEN** 같은 윈도우 내에서 동일한 회전 토큰을 가진 광고가 여러 번 수신된다
- **THEN** 시스템은 해당 광고를 1건으로만 집계한다

### Requirement: 하이파이브 성립 시 전용 햅틱
시스템은 하이파이브가 성립한 경우, 기존 득점·홈런 햅틱과 구분되는 전용 "하이파이브" 햅틱 패턴을 사용자의 워치에서 재생해야 한다(SHALL). 이 햅틱은 기존 득점 햅틱을 대체해서는 안 되며(MUST NOT), 기존 햅틱 직후 추가 재생되어야 한다.

#### Scenario: 하이파이브 성립 후 전용 패턴 재생
- **WHEN** 하이파이브가 성립으로 판정된다
- **THEN** 워치는 기존 득점 햅틱을 재생한 직후 전용 하이파이브 햅틱 패턴을 재생한다

#### Scenario: 매칭 실패 시 전용 패턴 없음
- **WHEN** 윈도우가 열렸지만 임계값을 만족하는 광고를 하나도 수신하지 못했다
- **THEN** 시스템은 전용 하이파이브 햅틱을 재생하지 않는다
- **AND** 기존 득점 햅틱은 영향 없이 정상 재생된다

### Requirement: 커뮤니티 탭 집계 배지
시스템은 하이파이브가 성립한 경우, 커뮤니티 탭 피드에 익명 집계 배지를 남겨야 한다(SHALL). 배지에는 하이파이브 상대의 식별자가 저장되어서는 안 되며(MUST NOT), 본인 기준의 카운트만 기록되어야 한다.

#### Scenario: 성립 시 배지 생성
- **WHEN** 윈도우 종료 시점에 하이파이브 성립 건수가 1 이상이다
- **THEN** 커뮤니티 탭 피드에 "해당 경기·해당 이벤트에서 N명과 하이파이브" 배지가 추가된다

#### Scenario: 상대 식별자 저장 금지
- **WHEN** 커뮤니티 탭 배지가 생성된다
- **THEN** 배지 레코드에는 하이파이브 상대의 계정 ID, 닉네임, 기기 식별자가 포함되지 않는다

#### Scenario: 성립 0건 시 배지 없음
- **WHEN** 윈도우 동안 하이파이브가 한 건도 성립하지 않았다
- **THEN** 커뮤니티 탭에 배지가 생성되지 않는다

### Requirement: 파일럿 팀 게이트
시스템은 본 기능을 파일럿 팀(LG, 두산) 사용자에게만 노출해야 한다(SHALL). 그 외 팀을 응원하는 사용자에게는 설정 토글이 표시되어서는 안 되며(MUST NOT), 어떠한 BLE 광고·스캔도 수행되어서는 안 된다(MUST NOT).

#### Scenario: 파일럿 팀 사용자에게 토글 노출
- **WHEN** 사용자의 응원 팀이 LG 또는 두산이다
- **THEN** 설정 화면에 "원격 하이파이브" 토글이 표시된다

#### Scenario: 비파일럿 팀 사용자에게 토글 숨김
- **WHEN** 사용자의 응원 팀이 LG, 두산이 아닌 다른 KBO 팀이다
- **THEN** 설정 화면에 "원격 하이파이브" 토글이 표시되지 않는다
- **AND** 시스템은 BLE 광고·스캔을 수행하지 않는다

#### Scenario: 원격 플래그 롤백
- **WHEN** 서버 측 파일럿 플래그가 off로 전환된다
- **THEN** 다음 앱 동기화 시 토글이 숨겨지고 진행 중이던 윈도우가 종료된다

### Requirement: 위치 raw 미수집 원칙
시스템은 본 기능을 제공하는 과정에서 사용자의 지리적 위치(GPS 좌표, IP 기반 위치, 기지국 기반 위치)를 수집, 저장, 전송해서는 안 된다(MUST NOT). 권한 요청 다이얼로그와 개인정보 처리방침에는 이 원칙이 사용자에게 명시되어야 한다.

#### Scenario: 위치 권한 요청 금지
- **WHEN** 사용자가 "원격 하이파이브" 토글을 켠다
- **THEN** 시스템은 위치 권한(iOS `CoreLocation`, Android `ACCESS_FINE_LOCATION` 등)을 요청하지 않는다

#### Scenario: 위치 raw 전송 금지
- **WHEN** 하이파이브가 성립하여 커뮤니티 배지가 생성된다
- **THEN** 배지 레코드나 광고 페이로드 어디에도 좌표·주소·기지국 식별자가 포함되지 않는다

#### Scenario: 권한 문구 명시
- **WHEN** 시스템이 블루투스 권한 요청 다이얼로그를 표시한다
- **THEN** 문구에는 위치 정보가 수집되지 않는다는 의미가 포함된다
