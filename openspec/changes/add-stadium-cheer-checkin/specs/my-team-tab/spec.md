## ADDED Requirements

### Requirement: 하단 탭 4번째 항목 추가
앱 하단 탭에 기존 `홈` / `상점` / `설정` 옆 4번째 항목으로 `내 팀` 탭이 등록되어야 한다(SHALL). 등록 자체는 코드에 들어가되, 활성화 명령이 떨어지기 전까지 진입점은 노출되지 않아야 한다.

#### Scenario: 다크 상태 미노출
- **WHEN** 활성화 명령 전이다
- **THEN** 사용자에게 `내 팀` 탭은 보이지 않는다
- **AND** 기존 3개 탭(`홈`, `상점`, `설정`) UI는 영향받지 않는다

#### Scenario: 활성화 후 노출
- **WHEN** 활성화 플래그/주석이 해제된다
- **THEN** 하단 탭 4번째 항목으로 `내 팀`이 노출된다
- **AND** 탭 탭하면 응원 랭킹 콘텐츠가 표시된다

### Requirement: 응원 랭킹 콘텐츠 표시
`내 팀` 탭은 1차 콘텐츠로 팀별 응원 누적 체크인 랭킹을 표시해야 한다(SHALL). 시즌·주간 토글과 사용자 본인 응원팀 순위 강조를 포함한다.

#### Scenario: 시즌/주간 토글
- **WHEN** 사용자가 시즌·주간 토글을 전환한다
- **THEN** 해당 단위의 팀별 순위가 표시된다

#### Scenario: 내 팀 위치 강조
- **WHEN** 사용자가 응원팀을 설정한 상태에서 랭킹을 본다
- **THEN** 본인 응원팀 행이 시각적으로 강조된다

### Requirement: 향후 콘텐츠 확장 자리 확보
`내 팀` 탭 컨테이너는 향후 팀별 뉴스 / 개인 체크인 달력 등 추가 콘텐츠를 임베드할 수 있는 구조여야 한다(SHALL). 현재 단계에서 뉴스·달력 화면을 구현하지는 않는다.

#### Scenario: 뉴스 자리 확장
- **WHEN** 후속 change에서 팀별 뉴스 화면이 추가된다
- **THEN** `내 팀` 탭 컨테이너 내부에 segmented control 또는 navigation으로 추가될 수 있다
- **AND** 기존 응원 랭킹 콘텐츠는 유지된다

#### Scenario: 개인 체크인 달력 자리 확장
- **WHEN** 후속 change에서 본인 체크인 이력을 달력 UI로 보여주는 화면이 추가된다
- **THEN** `내 팀` 탭 내부 segmented(랭킹 / 내 체크인 / 뉴스)로 진입점이 마련될 수 있다
- **AND** 백엔드 `GET /cheer-events/me` 엔드포인트와 `idx_cheer_events_user_clientts` 인덱스를 그대로 활용한다
