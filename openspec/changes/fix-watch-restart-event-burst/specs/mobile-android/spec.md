## ADDED Requirements

### Requirement: 워치 재연결 초기 이벤트 억제
모바일은 워치 동기화 스트림을 시작하거나 재연결할 때 수신한 과거 이벤트 스냅샷을 신규 워치 알림으로 전송하지 않아야 한다(MUST).

#### Scenario: 스트림 연결 직후 최근 이벤트 수신
- **GIVEN** 사용자가 이미 진행 중인 경기를 워치와 동기화한 상태
- **WHEN** 모바일이 경기 스트림에 연결되어 최근 이벤트 목록을 수신하면
- **THEN** 모바일은 해당 이벤트 목록을 기준 커서로만 반영하고 워치 햅틱 또는 영상 알림으로 전송하지 않는다

#### Scenario: 스트림 연결 이후 신규 이벤트 수신
- **GIVEN** 모바일이 초기 이벤트 목록을 기준 커서로 반영한 상태
- **WHEN** 이후 신규 경기 이벤트가 수신되면
- **THEN** 모바일은 사용자 설정에 따라 워치 햅틱 또는 영상 알림으로 전송한다

### Requirement: 워치 햅틱 이벤트 최신 항목 유지
모바일은 워치 햅틱 이벤트를 전송할 때 오래된 이벤트가 누적 재전달되지 않도록 최신 이벤트 기준으로 갱신해야 한다(MUST).

#### Scenario: 햅틱 이벤트 전송
- **WHEN** 모바일이 워치에 햅틱 이벤트를 전송하면
- **THEN** 워치가 이벤트의 최신성을 판단할 수 있는 발생 시각을 함께 전달한다

#### Scenario: 연속 햅틱 이벤트 전송
- **GIVEN** 모바일이 여러 경기 이벤트를 순서대로 수신한 상태
- **WHEN** 각 이벤트를 워치로 전송하면
- **THEN** 워치 Data Layer에는 과거 햅틱 이벤트가 무제한 누적되지 않고 최신 이벤트 기준으로 갱신된다
