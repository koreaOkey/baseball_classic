## ADDED Requirements

### Requirement: 오래된 햅틱 이벤트 무시
워치는 앱 재시작 또는 Data Layer 재전달로 오래된 햅틱 이벤트를 수신했을 때 진동, 화면 깨우기, 이벤트 영상을 실행하지 않아야 한다(MUST).

#### Scenario: 발생 시각이 오래된 햅틱 이벤트 수신
- **GIVEN** 워치가 모바일로부터 햅틱 이벤트를 수신할 수 있는 상태
- **WHEN** 수신한 햅틱 이벤트의 발생 시각이 freshness 기준을 초과하면
- **THEN** 워치는 해당 이벤트의 진동, 화면 깨우기, 이벤트 영상을 실행하지 않는다

#### Scenario: 기존 경로 기반 햅틱 이벤트 재전달
- **GIVEN** 워치 또는 Data Layer에 과거 햅틱 이벤트 항목이 남아 있는 상태
- **WHEN** 워치가 앱 재시작 후 해당 이벤트를 다시 수신하면
- **THEN** 워치는 경로에 포함된 시각 정보를 기준으로 오래된 이벤트를 무시한다

#### Scenario: 최신 햅틱 이벤트 수신
- **GIVEN** 워치가 모바일로부터 최신 경기 이벤트를 수신한 상태
- **WHEN** 해당 이벤트가 freshness 기준 안에 있고 이전에 처리한 커서보다 최신이면
- **THEN** 워치는 사용자 설정에 따라 진동, 화면 깨우기, 이벤트 영상을 실행한다
