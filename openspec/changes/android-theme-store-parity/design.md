## Context

Android 테마 상점이 가짜 포인트 목업으로만 구성되어 있었고, iOS는 리워드 광고 + Supabase 동기화 + 12개 실제 테마가 이미 운영 중이었다. 동일한 백엔드(Supabase `user_theme_purchases`, `user_settings` 테이블)를 공유하므로, Android도 같은 테이블 구조를 활용하여 크로스 플랫폼 동기화를 달성했다.

## Goals / Non-Goals

**Goals:**
- Android 테마 상점을 iOS와 동일한 기능 수준으로 끌어올림
- 리워드 광고(AdMob)를 통한 테마 잠금 해제 수익화 경로 확보
- 로그인 시 Supabase에서 테마 구매 이력/설정 자동 복원
- 워치 테마와 폰 테마 독립 관리 (워치 전용 스토어 테마)
- 워치 스토어 테마 동기화: 모바일에서 적용 시 워치에 즉시 반영

**Non-Goals:**
- Premium(인앱 결제) 테마 구현 (주석 처리, 향후 구현)
- FCM 푸시 알림 연동
- Android 위젯/Live Activity 구현
- Phone App 테마 탭 (준비 중 상태 유지)

## Decisions

### 1. 스토어 테마를 워치 앱에 내장 (vs. 동적 색상 전송)
- **선택**: 워치 앱에 12개 스토어 테마 정의 + 배경 이미지를 미리 포함, 모바일에서 `store_theme_id`만 전송
- **대안**: 모바일에서 ARGB 색상값을 Data Layer로 전송 → 워치에서 동적 테마 생성
- **이유**: 배경 이미지를 Data Layer Asset으로 전송하면 지연과 복잡도가 증가. iOS도 동일한 내장 방식 사용. 테마 추가 시 양쪽 앱 업데이트 필요하지만, 테마 변경 빈도가 낮아 허용 가능.

### 2. Watch 테마 ↔ Phone 테마 분리
- **선택**: `activeTheme`는 워치 전용, 폰 UI는 항상 `teamTheme`(응원팀 색상) 사용
- **이유**: iOS와 동일한 구조. Phone App 테마는 아직 "준비 중"이므로 폰 색상이 워치 테마에 영향받으면 안 됨.

### 3. 로컬 + 서버 이중 저장
- **선택**: `SharedPreferences`에 즉시 저장 + 로그인 상태면 Supabase에도 비동기 저장
- **이유**: 비로그인 사용자도 테마 이용 가능. 로그인 시 서버에서 복원하여 기기 간 동기화.

### 4. 이닝 텍스트 색상 처리
- **선택**: 기본 `Orange500` 하드코딩 유지, 선셋 오렌지/파이어 레드만 `Color.White`로 오버라이드
- **대안**: 모든 테마에서 accent 색상 사용
- **이유**: 대부분 테마에서 Orange500이 가독성 좋고 브랜드 톤에 맞음. 배경과 충돌하는 2개 테마만 예외 처리.

## Risks / Trade-offs

- **[테마 추가 시 양쪽 업데이트 필요]** → 테마 변경 빈도가 낮고, 동일 코드베이스 모노레포에서 관리되므로 허용 가능
- **[비로그인 테마 구매 데이터 유실]** → SharedPreferences 로컬 저장으로 기기 내 유지. 로그인 유도 UX는 향후 개선
- **[워치 앱 크기 증가]** → 배경 이미지 1개(야구가 좋아) 추가. 나머지는 코드 기반 그라데이션으로 경량화
