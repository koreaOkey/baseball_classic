## ADDED Requirements

### Requirement: 모바일→워치 스토어 테마 동기화
모바일에서 스토어 테마를 적용하면 Data Layer를 통해 워치에 store_theme_id를 전송하고, 워치 앱은 내장된 스토어 테마 정의로 UI를 즉시 변경해야 한다.

#### Scenario: 스토어 테마 적용 시 워치 동기화
- **WHEN** 모바일에서 스토어 테마를 적용
- **THEN** 워치 UI가 해당 스토어 테마의 색상과 배경으로 즉시 변경된다

#### Scenario: 기본 테마로 복원 시 워치 동기화
- **WHEN** 모바일에서 기본 팀 테마로 복원
- **THEN** 워치 UI가 응원팀 기본 색상으로 복원된다

### Requirement: 워치 배경 이미지 지원
배경 이미지가 있는 스토어 테마는 워치 라이브 게임 화면에 해당 이미지를 전체 배경으로 표시하고, 가독성을 위한 반투명 오버레이를 적용해야 한다.

#### Scenario: 배경 이미지 테마 적용
- **WHEN** 배경 이미지가 있는 스토어 테마(예: 야구가 좋아)가 적용
- **THEN** 워치 라이브 게임 화면에 해당 이미지가 전체 배경으로 표시되고 45% 검정 오버레이가 적용된다

### Requirement: 스토어 테마별 그라데이션 배경
배경 이미지가 없는 스토어 테마는 워치 라이브 게임 화면에 테마 색상 기반 그라데이션 배경을 표시해야 한다.

#### Scenario: 그라데이션 배경 테마 적용
- **WHEN** 배경 이미지가 없는 스토어 테마가 적용
- **THEN** 테마의 gradientStart → gradientEnd → Gray950 그라데이션이 배경에 표시된다

### Requirement: 가독성 기반 이닝 텍스트 색상
배경색과 이닝 텍스트 색상이 충돌하는 테마에서는 흰색으로 대체하여 가독성을 확보해야 한다.

#### Scenario: 선셋 오렌지/파이어 레드 테마의 이닝 텍스트
- **WHEN** 선셋 오렌지 또는 파이어 레드 테마가 적용
- **THEN** 이닝 텍스트가 흰색으로 표시된다

#### Scenario: 기타 테마의 이닝 텍스트
- **WHEN** 그 외 테마가 적용
- **THEN** 이닝 텍스트가 Orange500으로 표시된다
